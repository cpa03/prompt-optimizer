# Story 1.3: History Record Management

**Status**: Draft

## Story

> **As a** user,
> **I want** to view and manage my prompt optimization history,
> **so that** I can reuse previous optimizations and track my work over time.

## Acceptance Criteria

1. User can view optimization history sorted by timestamp
2. User can search and filter history records
3. User can reuse a previous prompt from history
4. User can delete individual history records
5. History records include original prompt, optimized result, and model used
6. History persists across sessions (local storage)
7. User can export history to JSON file
8. User can import history from JSON file (merge or replace option)

### Key Information

- **Data Models**:
  - HistoryRecord: `{ id: string, originalPrompt: string, optimizedPrompt: string, model: string, timestamp: Date, tags?: string[] }`
  - HistoryFilter: `{ searchText: string, dateRange: [Date, Date], models: string[] }`
  - [Source: packages/core/src/services/history/history.types.ts]

- **API Contracts**:
  - GET `/api/history` - List history with filters
  - GET `/api/history/:id` - Get single record
  - DELETE `/api/history/:id` - Delete record
  - POST `/api/history/export` - Export history
  - [Source: packages/core/src/services/history/history.service.ts]

- **Components**:
  - Use NaiveUI list and card components
  - Implement virtual scrolling for performance
  - Use modal for full details view
  - [Source: packages/ui/README.md]

- **Storage**:
  - Use localStorage for persistence
  - Implement data versioning for migrations
  - Max storage: 5MB per origin
  - Consider IndexedDB for larger datasets

- **Dependencies**:
  - @prompt-optimizer/core - history service
  - naive-ui - UI components
  - date-fns - date formatting
