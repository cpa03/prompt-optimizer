# Story 1.2: Model Management Configuration

**Status**: Draft

## Story

> **As a** user,
> **I want** to configure and manage multiple LLM models with their API keys,
> **so that** I can use different AI models for prompt optimization based on my needs.

## Acceptance Criteria

1. User can add new custom models with API endpoint and key
2. User can edit existing model configurations
3. User can delete model configurations
4. API keys are stored securely with encryption
5. User can test model connection before saving
6. System validates API key format before saving
7. Model list displays with visual status indicators (configured/unconfigured/invalid)

### Key Information

- **Data Models**:
  - ModelConfig: `{ id: string, name: string, provider: string, endpoint: string, apiKey: string, configured: boolean }`
  - ModelProvider: `{ id: string, name: string, icon: string, defaultEndpoint: string }`
  - [Source: packages/core/src/services/model/model.types.ts]

- **API Contracts**:
  - GET `/api/models` - List all models
  - POST `/api/models` - Create new model
  - PUT `/api/models/:id` - Update model
  - DELETE `/api/models/:id` - Delete model
  - POST `/api/models/:id/test` - Test model connection
  - [Source: packages/core/src/services/model/model.service.ts]

- **Components**:
  - Use NaiveUI form components
  - Implement form validation with visual feedback
  - Use modal dialogs for configuration
  - [Source: packages/ui/README.md]

- **Security Requirements**:
  - Encrypt API keys before storage
  - Use secure storage mechanism (localStorage with encryption)
  - Never log or expose API keys
  - Validate API key format before storage

- **Dependencies**:
  - @prompt-optimizer/core - model management service
  - crypto-js - encryption library
  - naive-ui - UI components
