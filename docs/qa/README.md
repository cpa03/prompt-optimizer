# QA Documentation

This directory contains Quality Assurance artifacts for the project.

## Directory Structure

- `gates/` - Quality gate decision files for stories
- `assessments/` - QA assessment reports (test design, traceability, risk profiles, NFR assessments)

## Quality Gate Files

Quality gate files are YAML documents that record the pass/fail decision for a story:

- **Location**: `gates/{epic}.{story}-{slug}.yml`
- **Schema Version**: 1
- **Possible States**: `PASS`, `CONCERNS`, `FAIL`, `WAIVED`

## Assessment Reports

Assessment reports provide detailed analysis:

- **Test Design**: `assessments/{epic}.{story}-test-design-{date}.md`
- **Requirements Traceability**: `assessments/{epic}.{story}-trace-{date}.md`
- **Risk Profile**: `assessments/{epic}.{story}-risk-{date}.md`
- **NFR Assessment**: `assessments/{epic}.{story}-nfr-{date}.md`

## Usage

QA artifacts are generated by the Test Architect & Quality Advisor agent when reviewing stories.

## Configuration

See `.bmad-core/core-config.yaml` for the `qa.qaLocation` setting.
